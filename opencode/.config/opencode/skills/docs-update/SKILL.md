# Documentation Update

Use this skill when updating documentation.

## When to Use

- Updating README files
- Adding or updating code comments
- Writing changelog entries
- Creating API documentation
- Adding inline documentation

## Workflow

1. Identify what needs documenting
2. Read existing docs for context
3. Make targeted updates
4. Show what changed

## Constraints

- Do NOT make code changes - only documentation
- Match existing documentation style
- Be concise - docs should add value, not length
- Update related docs together (e.g., README + CHANGELOG)
