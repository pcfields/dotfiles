# Bug Debugging

Use this skill when investigating and fixing bugs.

## When to Use

- Fixing runtime errors or crashes
- Addressing unexpected behavior
- Investigating logic errors
- Debugging test failures

## Workflow

1. **Rephrase the Bug**
   - Describe what should happen vs what is happening
   - Note any error messages or unexpected output

2. **Find Reproduction Path**
   - Identify steps to reproduce the issue
   - Note any conditions that trigger the bug

3. **Locate Likely Files**
   - Use error messages, stack traces
   - Search for relevant code paths

4. **Form Hypotheses**
   - Identify likely root cause
   - Propose a fix to test

5. **Test Hypotheses**
   - Make minimal edits to test fixes
   - Verify the fix works

## Constraints

- Don't guess - use evidence from code and errors
- Make one change at a time
- Test after each change
- Preserve working functionality
- Ask for clarification if root cause is unclear
