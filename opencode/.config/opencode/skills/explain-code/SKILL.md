# Explain Code

Use this skill when you need to understand code.

## When to Use

- Understanding unfamiliar code
- Explaining complex logic
- Describing what a function does
- Documenting existing code

## Workflow

1. Read the code thoroughly
2. Identify the main purpose
3. Trace the data flow
4. Explain in plain language

## Explanation Format

```
## Summary
[One sentence describing what this does]

## Inputs
[What it takes]

## Outputs
[What it returns]

## Key Points
- [Point 1]
- [Point 2]

## Side Effects
[Any side effects if applicable]
```

## Tips

- Use simple language
- Avoid jargon or explain it
- Focus on what matters
- Give examples if helpful
