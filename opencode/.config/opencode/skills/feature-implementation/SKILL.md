# Feature Implementation

Use this skill when adding new functionality to the codebase.

## When to Use

- Adding new endpoints or APIs
- Creating new UI components
- Adding new commands or scripts
- Building new integration

## Workflow

1. **Understand Requirements**
   - Read task and any acceptance criteria
   - Identify what the feature should do and not do

2. **Analyze Existing Code**
   - Find entrypoints and data flow
   - Identify similar existing patterns to follow

3. **Plan Implementation**
   - Propose small, ordered steps
   - Identify what files need changes

4. **Implement**
   - Follow the plan
   - Match existing code style and patterns

5. **Suggest Tests**
   - Propose focused test cases
   - Identify edge cases to consider

## Constraints

- Keep changes focused - implement one feature at a time
- Don't over-engineer - simple first, optimize later
- Use existing patterns before creating new ones
- Consider backward compatibility
